Overview

This is a simple JavaScript plugin for a hashchange-driven tabbed content switcher. Personally I'm not a tabsphiliac, but I needed those a few times recently and decided to make a semi-universal solution.

View on GitHub

Download v1.0

Features:

  • Vanilla JavaScript
  • SCSS stylesheet
  • Switching tabs via outside #links
  • Multiple simultaneous hashTabbers
  • Tested and working on those browsers:
    • Chrome 4+ (iOS 33+)
    • Firefox 4+
    • Opera 12+
    • Safari 5+ (iOS 7+)
    • IE 9+

Getting started

You will need those two files:

  • hashTabber.js – with a single object inside and four helper functions; so you could as well just paste it to your script.js to avoid an unnecessary HTML request
  • _hashTabber.scss – something like 20 lines of code that would be best to just @include in your global.scss

Build a html hashTabber section:

{% include examples/hashTabber-example-code-html.liquid %}

Important things for the whole thing to work are:

  • base class – the script will be looking for the all the elements with hashTabber class
  • relationships – the script assumes that .hashTabber has two immediate children: ol.hashTabber-nav (for navigation) and ol.hashTabber-data (for content) with their immediate children being lis; the script pairs up two lis with the same index from those two lists
  • the id – a HTML attribute id="distinct-name" that will be used to generate links for tabbed navigation
  • links – every navigation list element ol.hashTabber-nav > li needs to have an immediate child node <a> for #distinct-name=INDEX hrefs creation

Then just run the damn thing:

{% include examples/hashTabber-example-code-js-run.liquid %}

If the content changes after hashTabber was triggered, you can just use the .run() method again.

Options

It's rather pointless to change anything here, but if you like, please do:

{% include examples/hashTabber-example-code-js-options.liquid %}

For now, the only valuable options are:

  • data-defaultTab="3" – this HTML attribute of .hashTabber changes the default tab from 0 to the given number

It's also worth to mention, that both <ol> lists could contain countless different nodes (<h2> for example), as the script takes into account (and counts) only <li> nodes inside of them.
